# UI/UX設計方針

## フレームワーク

- **Wails v2** — Goバックエンド + Webフロントエンドのデスクトップアプリフレームワーク
- **React + TypeScript** — フロントエンド
- **Tailwind CSS** — スタイリング

## デザイン方針

- **ダークテーマ** をデフォルトとする（ゲームプレイ中に視認しやすい）
- シンプル・クリーンなレイアウト（余計な要素を排除）
- 翻訳テキストは **大きめのフォントサイズ**（最低16px、デフォルト18px）
- 読みやすさ優先（行間・余白を十分に確保）

## カラーパレット

```
背景:       #0f0f0f（ほぼ黒）
カード背景:  #1a1a1a
テキスト:    #e5e5e5（明るいグレー）
アクセント:  #6366f1（インディゴ）
エラー:      #ef4444（レッド）
成功:        #22c55e（グリーン）
```

## 画面構成

### メイン画面（翻訳表示）
- 上部: ステータスバー（キャプチャ中 / 翻訳中 / エラー）
- 中央: 翻訳テキストエリア（スクロール可能）
- 下部: 最終更新時刻

### 設定パネル（サイドパネルまたはモーダル）
- モニター選択（ドロップダウン）
- キャプチャインターバル（スライダー: 1〜30秒）
- 範囲選択（トグル + x/y/width/height入力）
- APIキー入力（マスク表示）

## インタラクション

- 設定変更は即座に反映（ページリロード不要）
- キャプチャ中はローディングインジケーターを表示
- エラー発生時はトースト通知で表示（自動消去5秒）
- 翻訳テキストが更新されたときは、スムーズにフェードインで表示

## アクセシビリティ

- フォントは日本語表示に適したものを選択（例: Noto Sans JP）
- テキストサイズはユーザーが調整可能にする（設定パネルで変更）
